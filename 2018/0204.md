[**点融区块链即服务实践**](http://ppt.geekbang.org/qconsh2017)
* 市场中存在的问题：传统的单点集中信息不够透明，存在效率问题
   * 完全依赖中央账本，信息来源和真实度缺乏透明性
   * 中央账本信息可能被更改而很难发现
   * 交易各方都需要事先建立信任
   * 只有一个中心信息节点，如果受影响，会影响整个系统
* 问题：信息不透明容易更改，交易成本高，依赖中心节点
* 解决方案：区块链通过分布式账本提高信息和交易安全性
   * 每个成员都拥有账本的完整拷贝
   * 如果一个成员的信息被篡改，会立即发现
   * 每笔交易需要达成共识，才能记录在账本上
* 优点：信息透明无法篡改，交易成本大幅降低，可以容忍节点故障
* 区块链使用中的问题
   * 技术门槛高
   * 安装部署复杂
   * 管理运维困难
   * 安全稳定性不高
   * 应用开发难度大
* BaaS：Blockchain as a Service

[**用超 7500 万的 GitHub 代码仓库实力解读：哪门编程语言热度最高？**](http://blog.csdn.net/csdnnews/article/details/79184674)
* 通过分析 GitHub Archive 上 7500w+ 个不同的仓库中超过 12.5 亿次的事件（包括推送新代码、fork、star、issue 处理等），每月汇总一次，计算了每种语言的 MAU(月活跃用户数量)。
* 排名：JS>PY>Java>C++>C>PHP>C#
* 用户群增长最快的语言分别有：Go，TypeScript，Kotlin 和 Rust
* 在过去的 7 年里，Ruby，PHP，Objective-C，CoffeeScript 和 Perl 在 GitHub 上的用户比例都显着下降
